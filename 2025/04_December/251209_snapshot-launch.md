# Snapshot V2 첫 공개 — 팀이 같은 언어를 갖기 시작한 날

그동안 준비해온 **Snapshot V2**를 팀원들에게 처음으로 공유한 날이다.  
단순히 “툴을 하나 만들었다”가 아니라, **팀의 업무 체계를 정렬하는 첫 규칙을 만든다**는 마음으로 데모를 진행했다.  

설명은 3단계로 이루어졌다.

1. **우리가 왜 스냅샷을 도입하는가 (Goal)**
2. **팀의 일이 실제로 어떤 흐름을 따라가는가 (Main Flow)**
3. **그 흐름을 어떻게 데이터화·정량화할 것인가 (Concept of Snapshot)**

실제 서비스 형태로 시연했고, 팀원들이 “이제 왜 필요한지 알겠다”, “앞으로 이게 더 진화할 것 같다”는 반응을 보였다.  
여기서 중요한 건 기능의 화려함이 아니라, **팀이 공통 언어를 갖기 시작했다**는 점이다.

## 본문

### 1) 목표를 먼저 다시 꺼내놓았다  
스냅샷의 목적은 팀의 상태를 보고서처럼 아카이브하는 게 아니다.  
**팀이 같은 프로젝트 구조와 목표 체계를 공유하게 만드는 것**, 그게 1차 목표다.

- Feature → Pre / Pair / Post
- Project → Module → Feature/Goal  
이 흐름을 한 장으로 그려줌으로써, 팀원들이 “우리가 하는 일이 이 구조 안에서 움직이고 있구나”를 직관적으로 이해했다.

사람들이 이 구조를 처음으로 “같이 본 순간”이 꽤 중요하다.  
개발/기획/디자인이 서로 다른 기준으로 일하는 문제를 해결하는 시작점이기 때문이다.


### 2) Main Flow를 보여주자, 반응이 바뀌었다  
프로젝트(MOTIIV, M-Desk, M-Connector…)  
→ 모듈(Spreadsheet, Article, Profile…)  
→ Feature/Goal

이 3단계를 UI로 흐르게 보여줬을 때 팀원들이 제일 많이 했던 말:

> “아 이게 이렇게 이어지는 흐름으로 보이네.”

평소 설명으로는 절대 전달되지 않던 구조가 **시각화되자 바로 이해됐다**.  
이건 팀 운영 측면에서 꽤 큰 전환점이다.

사람들은 늘 “본 대로” 이해한다. 그동안 말로만 설명하던 것보다 훨씬 설득력이 있었다.


### 3) Snapshot의 핵심 개념을 팀 기준으로 재정의했다  
서류처럼 쓰라고 하면 다들 귀찮아한다.  
그래서 Snapshot은 정말 최소 구조만 남겼다.

- Header  
  - Domain / Project / Module / Feature-Goal  
- past (week)  
  - Tasks  
  - Risk  
  - RiskLevel  
  - Collaborators (pair/pre/post)  
- this (week)  
  - Tasks

여기서 강조한 포인트는 두 가지였다.

1) **우리는 일의 ‘흐름’을 기록한다는 것**  
2) **Flow가 같아질 때 팀의 언어도 같아진다는 것**

실제 팀원들이 지난주/이번 주 데이터를 스냅샷 형태로 다시 보니,  
그동안 서로의 일을 온전히 파악하지 못했던 원인이 어디 있었는지 드러났다.  
각자 다른 기준으로 일의 범위를 잡고 있었던 것.


### 4) 시연 — “이거 신기하네”라는 반응의 의미  
데모 때 팀원들이 말한 “신기하다”, “오, 이거 괜찮다”는 건 단순한 호기심이 아니다.

그 말 뒤에는 이런 흐름이 깔려 있다.

- 지금까지 우리의 업무 방식이 정리된 형태로 존재한 적이 없었다  
- 누군가 주도적으로 구조를 잡아준 시도가 처음이다  
- 앞으로 이게 발전하면 업무 충돌이 줄어들고, 서로의 진행 상황을 더 명확히 볼 수 있겠다  

즉, **팀이 처음으로 방향성에 동의하기 시작했다**는 신호다.  
오늘은 단순한 설명회가 아니라, 팀이 앞으로 맞춰갈 기준점을 정한 날이다.


### 5) 오늘 느낀 점  
- 규칙을 만들었다기보다 **팀의 공통 언어를 정립하는 첫 접근**이었다.  
- 사람들이 구조를 이해하면, 피드백도 구조 안에서 나오기 때문에 훨씬 생산적이다.  
- 이 시도는 스냅샷 자체보다, 앞으로 팀이 “같이 일하는 방식”을 만들어가는 기반이 된다.  
- 무엇보다 중요한 건, **팀 분위기가 앞으로 더 좋아질 것 같다는 기대감**이 조금씩 생기고 있다는 것.
