# 페이지 하네스 편집기 도킹바·레이어·컨텍스트 메뉴 Figma식 폴리시

> Puck 기반 페이지 편집기의 크롬을 하루 통째로 Figma 감각에 맞춘 날. 새벽 꼬리 정리와 낮의 저장소 위생을 지나, 저녁엔 레이어 패널·QuickInserter·우클릭 컨텍스트 메뉴·하단 도킹바를 차례로 다듬으며 base-ui Popover의 닫힘·배타·앵커와 iframe 브라우징 컨텍스트 경계를 반복해서 되짚었다.

## 배경

컴포넌트 갤러리와 짝을 이루는 페이지 쪽, Puck 기반 페이지 하네스 편집기를 Figma식으로 계속 밀어붙이는 흐름의 하루였다. 전날까지 상호작용과 속성 패널을 손봤다면, 이날은 편집기의 크롬(둘레 UI) 자체가 주 대상이었다. 좌측 레이어 패널, 하단 캔버스 도킹바, 컴포넌트 삽입용 QuickInserter, 우클릭 컨텍스트 메뉴가 그 표면들이다.

시간축으로 보면 세 덩어리로 읽힌다. 자정 무렵엔 전날 작업의 꼬리를 정리했다. floating 표면(도킹바·툴바·줌)의 elevation을 복원하고, 크롬 타입 크기를 밀도와 무관하게 고정한 근거를 주석으로 남기고, CopyPromptButton의 idle 타이머가 언마운트 뒤에도 상태를 건드리던 것을 클린업했다. 낮 11시에서 13시 사이엔 저장소 위생을 손봤다. awl이 추적할 필요 없는 파일들과 오래된 AGENTS.md, 백업 데이터를 걷어내고 gitignore를 정리했다. 그리고 저녁 19시 56분부터 22시 15분까지가 본편이다. 이 세 시간 동안 레이어 패널부터 도킹바까지 편집기 크롬 거의 전부를 연달아 건드렸다.

하루를 관통하는 기술적 주제는 두 가지였다. 하나는 base-ui Popover의 생애주기다. 언제 닫히고(바깥 누름), 둘 이상이 동시에 뜨지 않게 하고(배타 오픈), 어디에 뜨는지(앵커)를 컨텍스트 메뉴와 QuickInserter를 오가며 반복해서 마주쳤다. 다른 하나는 iframe이 별도 브라우징 컨텍스트라 host document로 이벤트가 버블되지 않고 좌표계도 다르다는 경계였다. 좌클릭 닫힘이 발동하지 않는 것도, 노드 위치에 팝오버를 앵커하려면 좌표를 변환해야 하는 것도 같은 뿌리에서 나왔다.

## 핵심 내용

### 개별 작업 기록 (시간순)

- floating 표면(dock/toolbar/zoom) elevation 복원
  - 접힘 좌패널 pill, 줌 도킹바, 접힘 우툴바에 data-overlay DOM 표식을 달고 리치 elevation을 되살렸다. 좌패널 pill은 chrome-global의 리셋(하위 전체 box-shadow:none !important) 아래에 있어 !important와 specificity로 이겨야 했고, 접힘 topbar만 조건부 표식을 줘서 펼침 상태는 flush를 유지했다. 이 표식 덕에 lint의 카드중첩 오탐 3건이 data-overlay 게이트로 자연히 소거됐다.

- 크롬 타입 밀도-무관 고정 근거 주석
  - 밀도 프리셋을 바꿔도 크롬 텍스트 크기가 안 변하는 게 버그가 아니라 설계임을 문서와 주석으로 못박았다. 밀도표는 행 높이·컨트롤 높이·아이콘 버튼만 밀도별로 바꾸고 타입은 밀도 무관이라는 원칙이다. 크롬 타입을 12px로 통일하되 label/body/title이 11/12/13 리터럴을 유지하는데, 이걸 폰트 변수로 치환하면 11과 13이 12로 뭉개져 위계가 깨지므로 리터럴 유지가 정답이라는 근거를 표본 위치에 남겼다. 렌더는 무변경.

- CopyPromptButton idle 타이머 언마운트 클린업
  - 상세 라우트의 복사 버튼이 성공/실패 후 1.6초 뒤 상태를 idle로 되돌리는 setTimeout 두 곳을 useRef에 담아 매 클릭마다 이전 타이머를 clear하고 재설정하도록 하고, effect cleanup이 언마운트 시 clearTimeout하게 했다. 1.6초 안에 라우팅을 이탈해도 언마운트된 컴포넌트에 setState가 불리지 않는다. 언마운트 시 경고·pageerror가 0인지 확인하는 e2e도 함께 붙였다.

- 저장소 위생: 불필요한 추적파일 삭제
  - awl 설정 파일과 usability 워크아이템 이전에 떠 있던 백업 데이터 등 추적할 필요 없는 파일들을 정리했다. 리포지토리에 반복해서 딸려오던 잡음을 이 시점에 걷어냈다.

- 저장소 위생: 오류 내역 업데이트·gitignore 정리
  - gitignore를 손봐 awl이 추적할 필요 없는 경로를 제외하고, 오래되어 실효성이 없어진 AGENTS.md를 삭제했다. 낮 시간의 이 두 커밋으로 저녁 본편에 들어가기 전 작업 트리를 깨끗한 상태로 맞췄다.

- undo 토스트를 하단에서 상단 중앙으로 이동
  - 실행취소 토스트가 화면 하단에 떠서 줌 도킹바와 겹치던 것을 상단 중앙으로 올렸다. 진입 애니메이션 방향도 위에서 아래로 반전했고, arm 배너 리팩터로 부재가 된 죽은 CSS 규칙을 함께 제거했다. 저녁 크롬 폴리시의 첫 손질이었다.

- 레이어 패널 hover/선택 음영 제거
  - 레이어 행에 hover나 선택 시 깔리던 배경 음영을 걷어냈다. 대신 액션 버튼 hover는 배경 대신 아이콘 stroke만 진하게 해서 반응을 표현하도록 바꿨다. 무거운 배경 강조를 가벼운 stroke 신호로 대체하는 방향이 이날 시각 작업 전반의 기조가 됐다.

- QuickInserter 탭바 outline 박스 제거·트랙 리스타일
  - 탭바를 감싸던 outline 박스를 없애고, 트랙에 배경과 radius를 주고 활성 탭에는 카드면 배경을 입혔다. 네모난 테두리로 구획을 나누던 방식에서 배경 채움으로 활성 상태를 표현하는 방식으로 바꿨다.

- 레이어 긴 라벨 우측 fade-mask
  - 레이어 라벨이 길어 hover나 선택 시 우측 액션 아이콘 뒤로 텍스트가 삐져나오던 것을, 우측에 fade-mask를 걸어 아이콘 뒤 텍스트가 자연스럽게 사라지도록 처리했다. 잘림 표시 대신 페이드로 밀도를 살렸다.

- 컨텍스트 메뉴와 컴포넌트 추가 팝오버 배타 오픈
  - 우클릭 컨텍스트 메뉴와 루트 삽입 팝오버가 동시에 두 개 뜰 수 있던 것을 최상위 상태 전환에서 배타화했다. 삽입 팝오버를 열 때 메뉴 상태를 비우고 그 반대도 비우게 해서 한 번에 하나만 뜨게 했다. 같은 메뉴에 딸린 자식 서브팝오버 경로는 영향받지 않게 분리했다.

- QuickInserter 최근 사용을 별도 탭으로 분리
  - 컴포넌트 목록에 섞여 있던 최근 사용 항목을 별도 탭으로 떼어냈다. 자주 쓰는 것과 전체 목록을 탭으로 구분해 삽입 흐름을 짧게 했다.

- iframe 좌클릭 시 컨텍스트 메뉴·QuickInserter 닫힘 브리지
  - 이날 iframe 경계 문제의 대표 사례다. 캔버스(iframe) 좌클릭은 별도 브라우징 컨텍스트라 host document로 버블되지 않아, base-ui Popover의 바깥 누름 닫힘이 발동하지 않았다. iframe 클릭을 host의 frameElement로 재발행해 base-ui의 바깥 누름 판정을 그대로 태웠다. 열린 팝오버가 있을 때만 전달하고, Puck 오버레이의 stopPropagation을 피하려 capture 페이즈에서 들었다.

- 아트보드 리사이즈 핸들 더블클릭 fit-to-content
  - 아트보드 하단 리사이즈 핸들을 더블클릭하면 높이가 콘텐츠에 맞게 자동으로 줄거나 늘도록 했다. 수동 드래그로 맞추던 것을 한 번의 더블클릭으로 콘텐츠 높이에 스냅하게 만든 편의 기능이다.

- 레이어 행 LayerRow+React.memo 추출로 선택 전환 롱태스크 제거
  - 이날 유일한 성능 작업이자 가장 큰 diff였다. 레이어 패널 한 파일을 크게 리팩터해 각 행을 LayerRow 컴포넌트로 뽑고 React.memo로 감쌌다. 선택이 바뀔 때마다 패널 전체가 리렌더되며 롱태스크가 잡히던 것을, 선택 변화에 영향받는 행만 갱신되도록 바꿔 선택 전환 비용을 없앴다.

- 컨텍스트 메뉴 '자식 요소 추가' 강조 배경·항목 padding 축소
  - 컨텍스트 메뉴에서 '자식 요소 추가' 항목만 상시 배경으로 강조하고, hover 시엔 currentColor를 섞어 평상 상태와 더 뚜렷이 구분되게 했다. 메뉴 항목 padding을 6px에서 4px로 줄여 전체 밀도도 높였다. 자주 쓰는 진입점을 눈에 띄게 만든 것이다.

- 우측 속성 패널 첫 필드 위 여백 4px 추가
  - 좌측 레이어 패널과 리듬을 맞추려 우측 속성 패널 첫 필드 위에 4px 여백을 넣었다. 좌우 패널의 헤더에서 첫 행까지 시작선을 정렬해 시각적 리듬을 일치시켰다.

- 캔버스 도킹바 선택/손바닥 커서 모드 토글 추가
  - Figma식 하단 도킹바에 선택(포인터)과 손바닥(pan) 라디오 토글을 넣고, pan 모드에서 좌클릭 드래그로 캔버스를 이동할 수 있게 했다(가운데 버튼 팬 로직 재사용). 상호배타 라디오로 도구 모드 스토어를 두고, 캔버스 양쪽의 pointerdown 가드를 가운데 버튼이거나 pan 모드의 좌클릭까지 받도록 확장했다. 가운데 버튼 팬은 커서 모드와 무관하게 항상 동작하고, 선택 해제 판정은 select 모드로만 게이팅해 팬 드래그 중엔 선택이 유지되게 했다. iframe on/off 양쪽을 브라우저 실측으로 확인했다.

- 레이어 패널 '자식 요소 추가' 팝오버를 노드 캔버스 위치에 앵커
  - 또 하나의 iframe 좌표 문제다. 레이어 행을 우클릭해 '자식 요소 추가'를 누르면 QuickInserter가 우클릭한 사이드바 좌표가 아니라 선택된 노드가 실제 렌더된 캔버스 위치 근방에 뜨도록 고쳤다. 엔트리 문서(iframe on/off)에서 노드를 조회해 rect를 구하고, iframe이면 브리지와 동일한 iframe에서 host로의 좌표 변환을 적용했다. 뷰포트보다 큰 컨테이너는 앵커를 가시 영역으로 클램프해 팝오버가 화면 밖으로 안 나가게 하고, 노드를 못 찾거나 크기가 0이면 기존 우클릭 좌표로 폴백해 크래시를 막았다. 캔버스 직접 우클릭도 같은 진입점을 공유해 함께 갱신됐다.

- 캔버스 도킹바 빠른 삽입 그룹·'+' 버튼·줌 우측 재배치
  - Figma식 하단 도킹바에 6종 프리미티브(제목/본문/스택/행/그리드/프레임) 빠른 삽입 아이콘 그룹과 흰색 원형 '+' 버튼(QuickInserter 컴포넌트 탭 오픈)을 추가했다. 빠른 삽입은 선택 노드 뒤에(없으면 루트 끝) 즉시 삽입하고, Puck insert 리듀서라 히스토리에 기록돼 실행취소가 된다. 동시에 줌 컨트롤을 별도 바로 분리해서, 도구바를 3열 그리드(1fr auto 1fr)로 재편해 중앙 메인 도구바와 우측 줌이 좁은 캔버스에서도 겹치지 않게 배치했다. 그룹 사이 구분선은 기존 divider를 재사용했다.

- 컨텍스트 메뉴 '이동'·'그룹' hover 서브카테고리 신설
  - 평면으로 나열되던 이동(4개)·그룹(5개) 항목을 hover 서브 팝오버로 묶어 메뉴 길이를 줄였다. Popover 기반을 유지해 서브메뉴를 부모 팝업 subtree 안에 렌더했기 때문에 ESC나 바깥 클릭의 전체 닫힘을 부모 Popover가 그대로 처리한다. 키보드는 오른쪽 화살표로 진입, 왼쪽 화살표로 이탈, 서브 안에서 순환을 지원한다. 기존 항목·단축키 표기·강조·삭제 danger·자식 추가 트리거는 회귀 없이 유지했다.

## 정리

하루를 관통한 건 base-ui Popover와 iframe 경계라는 두 개의 반복 주제였다. 컨텍스트 메뉴와 QuickInserter를 오가며 팝오버의 세 가지 질문, 언제 닫히고(바깥 누름), 둘이 겹치지 않게 하고(배타 오픈), 어디에 뜨는지(앵커)를 계속 마주쳤다. 특히 닫힘과 앵커 두 건이 모두 iframe이 별도 브라우징 컨텍스트라는 같은 뿌리에서 나온 게 인상적이었다. 좌클릭이 host로 안 버블돼 닫힘이 안 걸리는 문제는 클릭을 frameElement로 재발행해 base-ui 판정에 그대로 태워 풀었고, 노드 위치에 팝오버를 앵커하는 문제는 iframe에서 host로의 좌표 변환을 붙이고 화면 밖 클램프와 폴백까지 얹어 안전하게 처리했다. 같은 경계를 두 각도에서 만나며, iframe을 다룰 땐 이벤트 전파와 좌표계 두 가지를 항상 함께 의심해야 한다는 감각이 자리 잡았다.

시각 폴리시의 방향성도 한 줄로 꿰어졌다. 무거운 배경 강조를 덜어내는 쪽이었다. 레이어 패널의 hover·선택 음영을 걷고 아이콘 stroke만 진하게, 긴 라벨은 잘림 대신 fade-mask, 탭바는 outline 박스 대신 배경 채움, floating 표면은 테두리 대신 elevation으로 바꿨다. 강조를 더하기보다 노이즈를 빼서 위계를 만드는 절제가 편집기 크롬 전반의 기조였고, 이건 Figma 같은 성숙한 편집기가 왜 그렇게 조용해 보이는지에 대한 답이기도 했다.

기능 면에선 하단 도킹바를 Figma식으로 재편한 게 하루의 뼈대였다. 커서 모드 토글, 빠른 삽입 그룹, 그리고 줌을 우측으로 분리하는 3열 그리드까지, 흩어져 있던 캔버스 조작을 도킹바 한 줄로 모았다. 여기에 성능은 레이어 행을 memo로 뜯어 선택 전환의 롱태스크를 없앤 한 방으로 챙겼다. 자정의 꼬리 정리와 낮의 저장소 위생으로 트리를 깨끗이 비운 뒤 저녁 세 시간을 크롬에 집중한, 흐름이 분명한 하루였다.
