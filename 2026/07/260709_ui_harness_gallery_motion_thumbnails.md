# ui-harness 갤러리 UX: 모션 언어·접근성 라운드·썸네일·⌘K 팔레트

> 갤러리를 눈으로 다듬는 대신 스페이싱 헌법과 단일 모션 언어라는 시스템으로 세우고, 웹리서치·심층감사로 뽑은 접근성·키보드·터치 결함을 여러 라운드로 잡고, 카드 썸네일 71개를 프레임에 맞게 백필하고, ⌘K 커맨드 팔레트로 브라우징을 승격시킨 하루.

## 배경

ui-harness 갤러리는 컴포넌트가 90종을 넘기면서, 만드는 일보다 "일관되게 보이고 잘 조작되게" 다듬는 일이 무게중심이 됐다. 이날은 이 다듬기를 눈대중이 아니라 시스템으로 하려 했다. 먼저 여백과 컨트롤 높이를 역할별 단일값으로 못박는 스페이싱 헌법을 세우고, 표면마다 제각각이던 트랜지션을 하나의 모션 언어(이징·듀레이션·유틸)로 통일했다. 그 위에서 웹리서치와 8차원 병렬 코드 감사로 뽑은 백로그를 접근성·키보드·URL·대비 여러 라운드로 실행했고, 매 라운드를 브라우저에서 실측했다. 여기에 미큐레이션 컴포넌트가 카드에서 풀 demo를 클립해 '깨진 파편'처럼 보이던 문제를 썸네일 71개 백필로 해소하고, 마지막으로 ⌘K 커맨드 팔레트를 도입해 어디서든 91개 컴포넌트로 퍼지 점프하게 만들었다.

## 핵심 내용

### 개별 작업 기록 (시간순)

- 갤러리 스페이싱 헌법 도입 (여백·컨트롤 높이 전면 통일)
  - 역할별 단일값(인라인 6, 그룹 8, 블록 12, 카드 16, 영역 24, 섹션 32)을 styles.css에 문서화하고 전 표면을 정렬했다. 헤더와 본문 거터를 통일하고, 히어로 상하를 광학균형으로 맞추고, 툴바·헤더 컨트롤을 32px 단일 티어로, 카드·행·리뷰 패널 패딩을 통일했다. 눈으로 조금씩 맞추던 여백을 '외울 필요 없는 헌법'으로 바꾼 시작점이었다.

- 갤러리 모션 언어 도입 (카드 그리드 transform + 전 표면 마이크로 인터랙션)
  - 이징(crisp·exit·inout·spring-pop), 듀레이션(instant부터 slower까지), 유틸(hover-lift·state-tint·icon-nudge·focus-ring·stagger-item)로 단일 모션 체계를 정의하고 전 표면에 일관 적용했다. 카드·행은 스프링 대신 진입 200·복귀 280의 듀레이션 비대칭으로 자연스러운 depth 리프트를 주고, 툴바 슬라이딩 인디케이터, 헤더 스크롤 elevation 같은 미세 인터랙션을 붙였다. prefers-reduced-motion 범용 가드는 end-event 보존을 위해 0으로 끄지 않고 0.01ms로 처리했다. 검증 중 필터 전환 시 컨테이너 리마운트가 유발하던 스켈레톤 flash도 함께 없앴다.

- 갤러리 인터랙션 2차 강화 (a11y·터치 엣지케이스 + UX 폴리시, 웹리서치 기반)
  - 마이크로 인터랙션 스펙·모션 a11y·포인터/터치·갤러리 UX·CSS 성능 다섯 웹리서치를 근거로 12라운드를 실측하며 진행했다. 포커스 링 3:1 대비를 canvas 픽셀로 실측해 확보하고, 스티키 헤더에 scroll-padding-top을 줘 포커스가 헤더에 가리지 않게 했다. 손수 CSS hover를 hover:hover 미디어로 게이팅해 터치 sticky hover를 없애고, 리빌 폴백을 뷰포트 기준에서 pointer-coarse와 focus-within으로 교체해 넓은 터치 기기와 키보드 리빌 동등성을 맞췄다.

- 갤러리 3차 강화 (키보드·URL·상태·대비 10라운드, 심층 감사 기반)
  - 8차원 병렬 코드 감사(9 에이전트)와 완결성 크리틱으로 의존성 순서 백로그를 뽑아 10라운드로 실행했다. 온보딩 모달을 APG Dialog 패턴(role·aria-modal·Esc·초기 포커스·미니 트랩·복원)으로 세우고, 검색에 role=search와 접근이름을 붙이고, q/sort/view를 filter와 같은 규칙으로 URL에 동기화해 딥링크와 뒤로가기 복원을 지원했다. 라우트 전환 시 본문 포커스·aria-live 고지·skip-to-content를 넣고, placeholder 대비를 canvas 측정으로 2.58에서 5.99까지 올렸다.

- 애드버서리 리뷰가 잡은 R1·R6 회귀 2건 수정
  - 모달 포커스 트랩 effect가 비메모 onClose 탓에 App 리렌더마다 재실행돼 포커스를 뺏던 것을, onCloseRef로 감아 deps를 open 전환으로 축소했다. URL 동기화 effect가 마운트 시 localStorage 취향을 URL로 승격해 딥링크를 오염시키던 것도 마운트 1회 스킵 가드로 막았다. 강화 라운드가 만든 회귀를 반증 리뷰가 되짚어 잡은 사례다.

- 갤러리 4차 강화 (공유 Dialog·치트시트·상태 폴리시·최근본 8라운드)
  - 포커스 트랩·Esc·복원을 DRY하게 묶은 공유 접근성 Dialog를 추출하고 OnboardingModal을 그 위로 이관했다. ? 단축키 치트시트, 상세 404를 다른 빈 상태와 통일, CopyButton press를 프리미티브 표준으로, prefers-reduced-transparency 폴백, 최근 본 컴포넌트(localStorage)를 검색 결과 없음 화면의 복구 링크로 붙였다. 카드 그리드 roving tabindex와 320px 리플로우는 선행 과제·환경 제약으로 보류했다.

- 애드버서리 리뷰가 잡은 R11 회귀 (스택 Dialog Esc가 둘 다 닫힘)
  - 공유 Dialog 추출 뒤 온보딩과 치트시트가 겹쳐 열린 상태에서 Esc 한 번에 둘 다 닫히던 회귀를, 모듈 레벨 오픈 스택을 두고 Esc·Tab 트랩을 스택 최상단에만 적용하는 표준 모달 스택으로 고쳤다. 실측으로 최상단만 닫히고 아래는 유지됨을 확인했다.

- GUIDE에 카드 썸네일 프리뷰 규칙 추가
  - 카드 썸네일(작은 정적 프리뷰)과 상세 히어로(리치한 demo 글랜스)의 역할을 분리·명시했다. demo를 카드에 재사용하면 실크기라 프레임에 클립돼 깨진 파편처럼 보이는 원인을 문서화하고, 카드 프리뷰 작성 원칙(정적·작게·대표성)과 '신규는 프리뷰 엔트리 필수' 하드룰을 세웠다. 대량 백필 전에 규칙부터 세운 순서였다.

- 카드 썸네일 백필 68개 (demo 클립 대신 프레임에 맞는 정적 프리뷰)
  - 큐레이션 프리뷰 34개 외 미큐레이션 컴포넌트가 카드에서 풀 demo를 160px 프레임에 클립하던 문제를, 71 에이전트 워크플로우로 각 컴포넌트 소스·demo를 정독해 작은 정적 대표 프리뷰를 authoring하는 방식으로 풀었다. 기존 34개와 오류 전파를 차단하려 previews.backfill.tsx로 격리하고 getPreview 우선순위를 잡았다. tsc는 통과하나 Vite 런타임에서 실패하던 stable 컴포넌트 오import 3건도 함께 교정했다.

- 카드 썸네일 백필 나머지 3개 (71/71 완성)
  - API가 특수해 제외했던 3개(Avatar 프리미티브 조합, SortableList render prop 타입, namespace-as-type)를 개별 수정해 추가하고 71개 전부를 완성했다. 실측으로 세 개 모두 프레임에 맞게 렌더되고 Vite 오류·placeholder 강등이 0임을 확인했다.

- 커맨드 팔레트 ⌘K로 어디서든 91개 컴포넌트 퍼지 점프
  - ⌘K를 검색 입력 포커스에서 전역 커맨드 팔레트로 승격시킨 획기적 브라우징 UX. 공유 Dialog 위에 combobox/listbox 패턴(aria-activedescendant)으로 짓고, name/keyword/intent 랭킹의 퍼지 검색, 방향키 이동, Enter 상세 점프, Esc 닫기를 붙였다. 검색어가 없으면 '최근 본'을 먼저 노출하고, 각 행에 카테고리·상태 배지를 얹었다. 인라인 검색은 '/'로 옮기고 Dialog에 hideClose prop을 추가했다.

- 카드 썸네일 프레이밍 (오버사이즈 데모를 상단정렬 + 하단 페이드로)
  - 카드가 preview를 demo 우선으로 쓰면서 큰 데모가 160px 프레임에 중앙정렬로 클립돼 '가운데 잘린 파편'처럼 보이던 문제를 완화했다. 썸네일 콘텐츠 래퍼에 하단으로 투명해지는 마스크와 상단정렬을 적용해, 큰 데모는 위에서 들여다본 peek처럼 아래로 부드럽게 사라지고 프레임에 온전히 들어오는 작은 프리뷰는 영향받지 않게 했다. 프레임 박스는 유지하고 마스크는 콘텐츠에만 걸었다.

## 정리

이날의 12개 작업은 갤러리 UX를 '눈으로 하나씩 고치는 일'에서 '시스템으로 결정하는 일'로 옮기는 흐름이었다. 스페이싱 헌법과 모션 언어를 먼저 세운 것은, 이후의 모든 표면 다듬기가 그 단일값·단일 이징을 참조하게 만들어 개별 판단을 줄이려는 포석이었다. 접근성 강화가 2차·3차·4차로 여러 차례 반복된 건, 한 번에 끝나는 일이 아니라 웹리서치와 심층 감사로 새 백로그를 계속 발굴하며 라운드로 소진하는 성격이기 때문이다.

인상 깊었던 건 강화와 반증 리뷰가 짝을 이뤄 돈 대목이다. 3차 강화가 R1·R6 회귀를, 4차 강화가 R11 스택 Dialog 회귀를 만들었고 그걸 애드버서리 리뷰가 되짚어 잡았다. 접근성을 위해 도입한 공유 Dialog·포커스 트랩 같은 추상화가 새 회귀의 온상이 될 수 있다는 걸 매번 실측으로 확인한 셈이다. 카드 썸네일은 '규칙(GUIDE) 먼저, 대량 백필(71개) 나중, 프레이밍 마감'이라는 순서로 진행해, 대량 작업 전에 기준을 세우는 습관이 어떻게 오류 전파를 막는지 보여줬다. 마지막 ⌘K 팔레트는 이 모든 정돈된 표면 위에서 브라우징 자체를 한 단계 올린 마침표였다.
