# 페이지 에디터 캔버스 비주얼 폴리시: 절제된 마감에서 화끈한 마이크로 인터랙션까지

> Puck 페이지 에디터 캔버스와 인스펙터를 딥리서치 백로그 R17에서 R28로 다듬은 날. 저위험 마감에서 시작해 사용자가 대담 실험을 허용하자 elevation·글래스·핸들 틱·choreography·3D hover·슬라이딩 탭까지 밀어붙이고, 접힘 핸들 위치 버그 둘을 앞뒤로 잡았다.

## 배경

컴포넌트 갤러리와 짝이 되는 페이지 쪽, 즉 Puck 기반 페이지 에디터의 캔버스와 인스펙터를 비주얼로 정돈하는 날이었다. 개선안은 모션과 타입/공간을 각각 파는 딥리서치 백로그에서 나왔고, 라운드는 R17부터 R28까지 이어졌다.

하루의 곡선이 뚜렷했다. 앞부분(R17에서 R19)은 딥리서치 백로그 중 절제 기조에 맞는 저위험 마감이었다. 카테고리 헤어라인, tabular-nums, 아이콘 stroke 통일 같은 정갈함, 다크에서 떠 보이던 인스펙터 인풋을 표면으로 가라앉히기, 노드 호버에 옅은 pre-highlight 링을 얹기까지, 어느 것도 크게 튀지 않는 마감이었다. 그러다 사용자가 "화끈하게" 밀어도 된다고 허용하면서 곡선이 꺾인다. 뒷부분(R20에서 R27)은 리치 elevation 토큰과 글래스모피즘, 오버슈트 스프링을 선택 피드백에까지 쓰는 대담한 배치로 채워졌다. 그리고 하루의 앞과 끝은 접힘 핸들의 위치 버그 둘이 감쌌다. 시작은 사이드바가 접힐 때 엣지 핸들이 앱바로 솟아오르던 버그였고, 끝은 divider에 붕 떠 있던 접기 핸들을 헤더에 도킹된 flat ghost로 재설계한 것이었다. 캔버스는 호버 상태를 스크린샷으로 잡기 어려워, 거의 모든 시각 변경을 테스트 요소 주입과 실측(광도·그림자 층수)으로 검증하는 루프가 반복됐다.

## 핵심 내용

### 개별 작업 기록 (시간순)

- 사이드바 접힘 시 엣지 핸들이 앱바로 튀어오르던 버그 (이슈4 후속)
  - SidebarHandles가 접힌 사이드바의 rect.top을 읽는데, 접히면 폭 0에 opacity 0이라 top이 불안정해져 엣지 펼치기 핸들이 패널 상단에서 앱바 높이로 솟아올랐다. top을 '열린' 사이드바(폭 20 초과)에서만 읽도록 고쳤다. 한쪽이 접히면 반대쪽 열린 사이드바에서 읽고, 둘 다 접히면 직전값을 유지한다. 그리드 컨테이너 top은 헤더 영역을 포함해 너무 높아 부적합했다. 좌우 각각 열림/접힘에서 핸들 위치가 안정됨을 실측으로 확인했다.

- 정갈·일관 마감: 헤어라인·tabular-nums·아이콘 stroke 통일 (R17)
  - 딥리서치 백로그의 저위험 3건이다. 팔레트 카테고리 eyebrow에 옵티컬 인셋 헤어라인을 넣어 섹션 리듬을 주고(거터 앞에서 멈춤), 치수 배지·스크럽·칩·숫자 인풋에 tabular-nums를 걸어 값이 바뀔 때 자릿수가 흔들리지 않게 했으며, 크롬 아이콘 stroke를 2에서 1.5로 통일해 하나의 드로잉셋 느낌을 냈다. 팔레트 썸네일 프리뷰는 스코프에서 제외해 무영향을 확인했다.

- 인스펙터 인풋을 가라앉은 웰로 (R18)
  - Puck 필드 인풋이 패널보다 밝게 떠 있었다. 특히 다크에서 흰색 15% 배경이 card 위로 부유해 스티커처럼 보였다. Figma/Linear 관습대로 인풋 배경을 가장 낮은 표면으로 내리고 inset 상단 그림자로 깊이를 줘 표면으로 가라앉게 했고, 포커스엔 inset과 ring 헤일로를 얹었다. 캔버스엔 해당 인풋이 0개라 크롬 인풋에만 안전 적용했고, 라이트/다크 광도를 실측해 부유가 해소됨을 확인했다.

- 캔버스 노드 호버 pre-highlight 링 (R19)
  - 기존엔 노드 호버 시 이름 배지만 뜨고 링이 없어 무엇을 선택하게 될지가 약했다. Figma/Webflow처럼 호버에 옅은 링을 주기 위해, 호버이면서 선택 상태가 아닐 때만 노드 박스를 채우는 inset 1px primary 링과 옅은 틴트를 얹었다. box-shadow라 reflow가 없고 pointer-events none이라 클릭에 무간섭이며, 선택 상태는 Puck 기본 아웃라인 그대로 둬 중복을 피했다.

- 리치 elevation 토큰 + 토스트 글래스 (R20, 화끈 배치)
  - 테마 적응형 elevation 토큰을 정의했다. 라이트는 부드러운 다층 그림자, 다크는 상단 inset 하이라이트와 깊은 앰비언트(Geist 관습)다. :root 정의라 body 포털 표면(토스트·팝오버)에도 닿는다. 플로팅 표면에 적용해 진짜로 떠오르게 했고, 토스트엔 반투명 82%와 backdrop-blur, saturate로 글래스를 입혔다(Linear/Vercel 시그니처). 여기서부터 '화끈하게' 기조가 들어온다.

- 선택 노드 4코너 핸들 틱 + 스프링 팝 (R21)
  - 선택 시 노드 4모서리에 디자인툴 시그니처 핸들 틱(primary에 배경 테두리)을 얹고 스프링으로 탁 나타나게 했다. Puck 기본 아웃라인과 결합해 선택이 확실히 도구스럽게 읽힌다. 사용자가 대담 실험을 허용해서, 오버슈트 스프링을 선택 피드백에까지 끌어다 쓴 지점이다.

- 인스펙터 진입 choreography (R22)
  - FieldsOverride를 선택자 key 래퍼로 리팩터해, 선택이 바뀔 때마다 패널이 리마운트되며 진입 애니메이션(opacity와 translateY 7px)이 재생되게 했다. "이 패널이 이제 그 노드의 것"이라는 신호다(Figma/Framer 관습). transform과 opacity만 써 GPU 합성이고 reduced-motion은 범용 블록이 커버한다. 겸사겸사 3분기 early-return을 단일 return으로 정리했다(동작 동일).

- 정갈한 빈 상태 EmptyHint (R23)
  - 검색 no-result와 빈 레이어 트리의 밋밋한 텍스트를, 아이콘 칩과 제목과 한 줄 힌트로 된 공용 EmptyHint로 격상했다(일러스트 없이). 빈 패널이 고장이 아니라 의도된 여백으로 읽히게 하려는 것이다(Linear/Figma). 앞선 인스펙터 choreography와 결이 맞게 fade-in도 걸었다.

- 3D 마이크로 인터랙션: 카드·핸들 (R24, frontend-design 규칙)
  - 팔레트/섹션 카드 호버에 살짝 뜨는 translate3d와 scale, primary 틴트 글로우 그림자를 걸고 active엔 눌리는 축소를 줬다. 미세 오버슈트 이징으로 살아있는 질감을 냈다. 패널 핸들도 호버/액티브에 통통 튀는 촉감을 줬다. 사용자 지시(frontend-design 가이드 fallback 주입)로, 절제를 넘어 미세 3D hover로 인터페이스에 생기를 넣은 지점이다. translate3d라 GPU 합성이고 reduced-motion은 범용 블록이 커버한다.

- 치트 모달 글래스모피즘 + 투명도 축소 opt-out (R25)
  - 토스트에 이어 단축키 치트 모달에도 글래스(backdrop-blur와 88% 반투명, 리치 elevation)를 입혔다. 다만 prefers-reduced-transparency일 때는 토스트·치트 글래스를 불투명 단색으로 대체해 가독성과 전정 부담을 낮췄다(기존 헤더 opt-out과 일관). 컨텍스트 메뉴는 팝오버 래퍼 배경 이슈로 보류했다.

- 모든 크롬 버튼에 미세 3D hover (R26)
  - 헤더 토글, Puck 버튼, IconButton에 호버 리프트와 active 축소를 걸어, 카드에 이어 버튼까지 일관된 눌리는 질감으로 확장했다. 저장하기 CTA는 호버 시 primary 글로우로 강조했다. 편집기 전 인터랙티브 표면이 하나의 촉감으로 묶였다.

- 슬라이딩 탭 언더라인 (R27)
  - 팔레트 컴포넌트/섹션 탭의 정적 border-bottom을, 활성 탭으로 미끄러지는 하나의 언더라인 span으로 바꿨다. 활성 탭의 offsetLeft와 offsetWidth를 useLayoutEffect로 실측해(가변폭 대응) left/width를 트랜지션했고, 미세 primary 글로우를 얹었다. 탭 전환이 프리미엄 톤을 얻었다.

- 접기 핸들을 divider 부유 pill에서 패널 헤더 도킹 ghost로 (R28)
  - 사용자 피드백은 divider에 그림자 달린 pill로 붕 떠 있어 '말도 안 되는 위치'라는 것이었다. 헤더 액션처럼 각 패널 헤더 코너에 도킹된 flat ghost(투명 배경·무그림자, 호버 시 subtle 배경)로 재설계했다. 접힘 시엔 화면 엣지에 눈에 띄는 탭으로 전환해 펼치기 어포던스를 유지했다. 리로드 후 양 핸들이 헤더 코너에 도킹돼 부유감이 사라졌음을 확인했다.

## 정리

이날 열세 개 커밋을 관통하는 축은 "페이지 에디터 캔버스를 절제와 대담 사이에서 조율한다"였다. 하루의 곡선이 그 조율을 그대로 보여준다. R17에서 R19까지는 딥리서치 백로그 중 저위험만 골라 헤어라인, tabular-nums, 가라앉은 인풋, 옅은 호버 링 같은 마감을 얹었다. 어느 것도 크게 튀지 않았다. 그러다 사용자가 "화끈하게"를 허용하면서 R20부터 리치 elevation, 글래스모피즘, 오버슈트 스프링을 선택 피드백에까지 끌어다 쓰는 대담한 배치로 넘어갔다. 절제냐 대담이냐를 내가 임의로 정하지 않고 사용자 허용을 경계로 삼은 게, 뒤로 갈수록 과감해지면서도 근거를 잃지 않은 이유였다.

가장 배운 게 컸던 건 검증 방식이다. 캔버스의 호버·선택·진입 애니메이션은 스크린샷으로 잡기가 어렵다. 그래서 거의 모든 시각 변경을 테스트 요소 주입으로 확인했다. pre-highlight 링은 노드 박스에 테스트 링을 주입해 정확히 채우는지, 4코너 핸들 틱은 테스트 박스로 네 모서리 배치와 pop을, 가라앉은 인풋은 라이트/다크 광도를 숫자로, 토스트 글래스는 그림자 층수와 backdrop-blur 값을 실측했다. 눈으로 보기 어려운 상태일수록 "무엇을 측정하면 이게 맞다고 할 수 있는가"를 먼저 정하는 습관이, 화끈한 배치를 근거 없는 취향으로 흐르지 않게 잡아줬다.

그리고 하루를 감싼 두 개의 접힘 핸들 버그가 대비를 이룬다. 아침엔 접힌 사이드바에서 rect.top을 읽어 핸들이 앱바로 솟는 좌표 계산 버그였고, 저녁엔 divider에 떠 있어 '말도 안 되는 위치'라던 핸들을 헤더 도킹 ghost로 재설계한 배치 버그였다. 하나는 접힌 요소의 불안정한 값을 읽지 말고 열린 요소에서 읽으라는 계산의 교훈, 다른 하나는 부유하는 요소는 어딘가에 도킹시켜 근거를 줘야 한다는 배치의 교훈이었다. 화려한 마이크로 인터랙션을 잔뜩 얹은 날의 처음과 끝이 결국 "핸들이 어디에 붙어 있어야 하는가"라는 가장 기본적인 위치 문제였다는 게, 폴리시의 완성도가 화끈함이 아니라 어긋난 좌표 하나에서 갈린다는 걸 다시 일러줬다.
