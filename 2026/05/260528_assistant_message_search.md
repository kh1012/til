# 어시스턴트 메시지 검색 기능과 검색창 끊김 잡기

> 스레드 내/전체 메시지 검색을 붙이고, 검색창의 폭 애니메이션·드롭다운에서 생기던 첫 페인트 스파이크와 매 프레임 리렌더 끊김을 합성 레이어 유지/관찰 범위 축소/setState 스킵으로 걷어낸 하루.

## 배경

어시스턴트 위젯에 메시지 검색을 붙이는 작업이었다. 기능 자체(스레드 내/전체 검색)를 넣는 것에서 출발했지만, 검색 입력이 포커스 때 폭이 늘어나는 인터랙션과 body로 portal하는 드롭다운이 얽히면서 첫 페인트 스파이크와 매 프레임 리렌더 같은 끊김이 연달아 드러났다. 뒤 두 커밋은 그 끊김을 합성 레이어 유지와 불필요 재계산 제거로 다듬는 과정이다.

## 핵심 내용

### 개별 커밋 기록 (시간순)

- `bb3250155` feat(assistant): 스레드 내/전체 메시지 검색 기능 추가
  - 현재 스레드 내 검색과 프로젝트 전체 검색 두 모드를 제공한다. 백엔드 프록시 라우트는 기존 것을 그대로 쓰고, 클라이언트가 `thread_id` 쿼리로 스코프를 분기한다. OpenAPI 스펙엔 아직 없어 URL 빌더로 직접 전달했다.
  - 검색 UI는 오버레이/헤더 팝오버 안을 거쳐, 헤더 우측의 슬림 입력 필드와 body로 portal되는 드롭다운으로 확정했다. 입력은 포커스 시 280에서 320px로 늘되 테두리 변화는 없고, `max-w`로 패널 리사이즈 시 깨지지 않는다.
  - 드롭다운은 패널 폭을 따르지 않고 입력의 1.5배(480px) 고정이다. 2배는 슬림 입력 대비 과해 1.5배를 택했고, portal + 위치 실시간 계산으로 패널 overflow/리사이즈에 영향받지 않게 했다.
  - 결과 클릭 시 해당 스레드로 전환하고 메시지로 점프해 연한 accent로 잠깐 강조한다(보더 없음). 검색 매칭어는 글자만 accent, 탭 활성도 accent 채움.
  - blur 후 재오픈 시 결과 높이가 전체에서 max로 줄어 보이던 글리치를, 언마운트 대신 visibility 토글(항상 마운트로 높이 보존)로 해결하고 닫힘 동안엔 `inert`로 포커스/클릭을 차단했다. 디바운스 훅(`use-debounce`)과 query-key 구성도 함께 정리.

- `8de6b02a3` Assistant 검색창 성능개선
  - (본문 없음, diff 기반) 드롭다운 위치 계산에서 `setPos`를 값이 같으면 건너뛰도록 바꿨다. 검색창 폭 애니메이션 중 프레임마다 동일 좌표로 setState가 돌며 리렌더되던 끊김을 막는다.
  - `ResizeObserver`가 입력 박스(anchor)까지 관찰하던 걸 패널만 관찰하도록 줄였다. 입력은 우측 정렬이라 폭이 280→320으로 변해도 기준점인 우측 끝이 고정이라 재계산이 불필요한데, 관찰하면 폭 애니메이션 내내 매 프레임 콜백이 돌아 끊김을 만들었다.

- `d1ba37b35` feat(assistant): 검색·로딩 UI 다듬기 및 입력 맥락 태그 추가
  - 검색 입력은 `bg-muted` 바탕에 포커스 시 흰 배경을 opacity로 페이드인하고 폭을 280→320으로 늘린다. 드롭다운은 패널 폭과 무관한 고정 480px(입력 1.5배)로 body에 portal하고, 닫힘 땐 opacity로만 숨겨 합성 레이어를 유지했다. 이걸로 포커스 시 폭 애니메이션 끊김(첫 페인트 스파이크)을 없앴다.
  - '전체' 검색 결과에 스레드 구분선(`divide-y`)을 추가했다.
  - 색상: 이 디자인 시스템에서 accent는 흰/검정이라, 검색 매칭어는 파란 시맨틱 토큰 `accent-steel`을 폰트 색으로 쓴다. 메시지 점프 하이라이트는 연한 회색(`muted`)을 버블보다 12px 확장한 decorator 엘리먼트로 깔아(레이아웃을 밀지 않음) radius 0으로 표시한다.
  - 아바타: ring과 로고 간격을 넓히는 `loose` 옵션(`inset-2`)을 추가하고, 대기→스트리밍 전환 시 크기가 점프하지 않도록 대기 버블 아바타를 메시지 버블과 동일(`size-10 loose`)하게 맞췄다.
  - 입력 맥락 태그(프로젝트 아바타+이름)를 헤더 scope 뱃지 대신 Composer 위로 옮겼다.
  - smart-close: `hoverLock`을 구독하지 않고 store에서 명령형으로 읽어, lock 변경(검색 focus/blur, popover 열림)마다 패널이 리렌더되던 것을 제거했다.

## 정리

기능을 붙인 첫 커밋과 달리, 이날의 진짜 주제는 **"검색창의 끊김은 거의 다 합성/리렌더 문제"**였다.

첫 페인트 스파이크의 핵심은 합성 레이어 유지였다. 드롭다운을 닫을 때 언마운트하거나 display로 숨기면, 다시 열 때 레이어가 새로 만들어지면서 포커스 시 폭 애니메이션 첫 프레임이 튄다. 닫힘을 opacity로만 처리해 레이어를 살려 두니(`d1ba37b35`) 스파이크가 사라졌다. 같은 결로, blur 후 재오픈 시 높이가 줄어 보이던 글리치(`bb3250155`)도 언마운트 대신 visibility 토글로 항상 마운트해 높이를 보존하고 `inert`로 상호작용만 막는 방식이 답이었다. 보이는 것을 숨기되 DOM/레이어는 살려 둔다는 한 가지 원칙이 두 글리치를 같이 잡았다.

매 프레임 끊김은 불필요한 재계산/리렌더를 끊는 문제였다(`8de6b02a3`). 우측 정렬 입력은 폭이 늘어도 기준점인 우측 끝이 안 움직이므로 위치 재계산이 필요 없는데, anchor까지 `ResizeObserver`로 관찰하면 폭 애니메이션 내내 콜백이 돈다. 관찰 대상을 패널로만 좁히고, 좌표가 같으면 `setPos`를 건너뛰게 한 두 조치가 핵심이었다. smart-close에서 `hoverLock`을 구독 대신 store에서 명령형으로 읽은 것(`d1ba37b35`)도 같은 종류로, lock이 바뀔 때마다 패널 전체가 리렌더되던 걸 끊었다. "이 값 변화가 정말 리렌더를 일으켜야 하나"를 매번 되묻는 게 끊김 디버깅의 출발점이라는 걸 다시 새겼다.

마지막으로 색상은 디자인 시스템 제약과의 타협이었다. accent가 흰/검정뿐인 모노크롬 체계라 검색 매칭어 강조에 쓸 색이 없어, 파란 시맨틱 토큰 `accent-steel`을 글자 색으로만 빌려 썼다. 점프 하이라이트도 버블 배경을 바꾸는 대신, 버블보다 12px 큰 decorator를 뒤에 깔아 레이아웃을 밀지 않게 한 게 포인트다. 강조는 콘텐츠 흐름을 건드리지 말아야 한다는 원칙이 여기서도 통했다.
